# Contributing to Display Profile Manager

First off, thank you for considering contributing to Display Profile Manager! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
*   Check the [Issues](https://github.com/izu-x/DisplayProfileManager/issues) to see if the bug has already been reported.
*   If not, open a new issue. Use a clear title and provide as much context as possible (Windows version, monitor setup, logs from `%AppData%\DisplayProfileManager\Logs\`).

### Suggesting Enhancements
*   Open an issue with the "feature request" template.
*   Explain why this enhancement would be useful and how it should work.

### Your First Code Contribution
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  If you've changed APIs, update the documentation.
4.  Ensure the build passes (GitHub Actions will check this).
5.  Issue that pull request!

## Styleguide

### C# Coding Standards
*   Use PascalCase for classes, methods, and properties.
*   Use camelCase for private fields (with an underscore prefix, e.g., `_myField`).
*   Follow the existing formatting and structure in the project.

### Commit Messages
*   Use the [Conventional Commits](https://www.conventionalcommits.org/) format:
    *   `feat: add new feature`
    *   `fix: resolve a bug`
    *   `docs: update documentation`
    *   `ci: update build scripts`

## Questions?
Feel free to open an issue with the "question" label!
